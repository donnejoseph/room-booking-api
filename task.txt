Разработать REST API для системы бронирования переговорных комнат в офисе.
Система должна позволять пользователям:
●​ просматривать доступные комнаты,​
●​ бронировать свободные слоты,​
●​ управлять своими бронированиями.​
Основные сущности
User
●​ Использовать стандартную модель Django User (можно расширить при
необходимости).​
Room
●​ Название​
●​ Вместимость​
●​ Этаж​
Booking
●​ Пользователь (владелец брони)​
●​ Комната​
●​ Дата​
●​ Время начала​
●​ Время окончания​
Требуемый функционал
1.​ Аутентификация и регистрация пользователей​
○​ Использовать JWT или другую систему токенов.​○​ Защита ручек, доступных только авторизованным пользователям.​
2.​ CRUD для переговорных комнат​
○​ Только администратор может создавать, редактировать и удалять
комнаты.​
○​ Все пользователи могут просматривать список и детали.​
3.​ Бронирование комнат​
○​ Пользователь может бронировать комнату на конкретное время.​
○​ Нельзя бронировать уже занятую на конкретное время комнату.​
○​ Один пользователь не может бронировать более одной комнаты на одно
и то же время.​
4.​ Просмотр свободных комнат​
○​ Возможность фильтровать комнаты по дате, времени, вместимости и
этажу.​
5.​ Просмотр и управление бронированиями​
○​ Пользователь видит только свои бронирования.​
○​ Администратор видит все бронирования.​
Технологии
●​ Django 4.x​
●​ Django REST Framework​
●​ PostgreSQL​
●​ Docker (по желанию)​
●​ Pytest или стандартные тесты Django (покрытие основных ручек)​
Дополнительные требования●​ API-документация (Swagger или ReDoc).​
●​ README с описанием проекта и инструкцией по запуску.​
●​ Приложение должно быть готово к большому набору данных​
●​ Чистота кода, читаемость, соблюдение PEP8.​
●​ Корректная обработка ошибок и валидация данных.
